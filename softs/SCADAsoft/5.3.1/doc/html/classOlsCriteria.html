Detailed Description

This class is one of the class of the OLS (Object List Server) SCADAsoft component.

This class is used to filter the data in case of reading or subscribing request on a data list. For each such request, the client can specify a filter. A filter is composed of a field list and contains also a criteria (see OlsFilter class). A criteria is a logical combination (NOT, AND or OR) of elementary conditions than the data must fulfill for it to be part of the request result. An elementary condition is made of a field name, an operation and a comparison value. The OlsCriteria class is the mother class of OlsAndCriteria, OlsOrCriteria, OlsNotCriteria and OlsFieldCriteria classes. An OlsCriteria represents SQL like WHERE clause. For example: "(Color == red OR Color == blue) AND Size > 10"

static OlsCriteria* OlsCriteria::Parse ( const char *  criteria) [static]
Description:
this function parses the string argument and produces a criteria tree. The string grammar is a simplified SQL WHERE clause. 
criteria ::= field_name operation value
           | criteria AND criteria
           | criteria OR criteria
           | NOT criteria
           | ( criteria )
operation ::= >= | > | <= | < | == | != | & | | | LIKE
field_name ::= identifier
value ::= numerical_constant | string_constant
The LIKE operator allows to match a string field whith a reguler expression.
Parameters:
criteria[in]: the input string criteria.
Return values:
The criteria object.
