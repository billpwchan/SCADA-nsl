Detailed Description

This class is one of the class of the OLS (Object List Server) SCADAsoft component.

This class is used to store data elements. An object (object of OlsData class) represents a set of basic data (object of OlsField class). An object list (object of OlsList class) groups a set of objects containing the same kind of information. An object of OlsList class contains a dataset containing the fields of the elements of the list, a list of filters, a list of subscribers and the data elements list.

Description:
constructor. A list contains a set of data. Each data have the same fields called the fields of the list. The list manages subscriptions. The period is the period between two subscription refresh timeouts. The period is in milliseconds. The withHash parameter specifies if the list use (value set to 1) or not (value set to 0) a hash table for the searchs into the data elements list.
Parameters:
name[in]: name of the list,
period[in]: period between two subscription refresh timeouts,
withHash[in]: use a hash table (if set to 1) or not,
withLock[in]: use a lock for the modifications (if set to 1) or not,
withThreadHandle[in]: if set to 1, notifications are done in a separate thread,
startPeriodicUpdate[in]: if set to 0, the periodical timer for the notifications is not started at the creation of the list. It could be started using the startPeriodicUpdate() method.
Return values:
none.

int &  subscriptionId,
const OlsCallback  notificationCallback,
void *  arg = 0,
long  firstNotficationSize = -1 
) [virtual]
Description:
this function subscribes the caller to the modifications on the data matching the filter. All the matching data which are added to the list, modified or removed from the list are notified to the subscribed client using the notification callback. If a name is specified in the filter, then the subscribe method will try to retrieve a registered filter. Otherwise, the subscribe method will apply the filter as is.
Parameters:
filter[in]: filter,
subscriptionId[out]: subscription identifier,
notificationCallback[in]: user notification callback function,
arg[in]: user optional argument passed to the notification callback when called.
firstNotficationSize[in]: first notification data count (-1 for all data).
Returns:
  • ScsValid : success.
  • OlsBadFilterError : in case an unregistered name is specified for the filter.
