Detailed Description

This class provides a remote access to OlsList objects.

This class is used to remotely access the services of OlsList objects managed by a distant server process (except the modification service). The methods of this class are exactly the same than thoses of OlsList class, except than they have in first parameter the name of the list. 

 #include "ols.h"

ScsPriorityLevels  priority = SCS_PRIORITY_NORMAL 
)
Description:
constructor. This function creates an interface object for remote OlsList objects. It creates a connection to the server which manages the remote list objects. This server is identified by the logicalEnv and serverName parameters. For example, to access the services of an alarm list managed by an alarm server "AlmServer" running in the physical environment "ENV1_cactus", call the current constructor like this : 
 OlsServer *olsServer = new OlsServer("ENV1", "AlmServer");
Parameters:
logicalEnv[in]: name of the logical SCADAsoft environment in which the server managing the remote OlsList objects is running,
serverName[in]: name of the server which manages the OlsList objects.
priority[in]: priority of the messages to the OlsServer (only useful with redudancy).
Return values:
none. The result of the connection can be obtain by calling the getStatus() function.

ScsStatus OlsServer::readAscending ( const char *  listName,
const OlsFilter &  filter,
int  startId,
int  max,
OlsDataSet *&  result 
) const
Description:
read in the remote list all the data matching the filter. The search starts from the data which key is startId and ends either if the end of the list is reached, or if max data have been found in the list. If startId equals -1, then the search starts at the begining of the list. If max equals -1, then the search ends at the end of the list. If a name is specified in the filter, then the read method will try to retrieve a registered filter. Otherwise, the read method will apply the filter as is.
Parameters:
listName[in]: name of the remote list,
filter[in]: filter object,
startId[in]: key of starting element,
max[in]: maximum number of elements to read,
result[out]: set of OlsData objects containing the read values.
Returns:
  • ScsValid : success.
  • OlsBadListNameError : the remote server does not export the specified list.
  • OlsErrHostServer : cannot connect to the ascmanager process.
  • OlsBadFilterError : in case an unregistered name is specified for the filter.
  • OlsBadKeyError : in case an unknown key is given in startId.
  • OlsORBError : communication failure with the remote server.
