Detailed Description

The ScsTimer class provides a mean of calling asynchrounously a callback function. 

 #include "scstimer.h"

typedef enum ScsTimer::KindOfTime ScsTimer::KindOfTime
Description:
enum corresponding to the kind of time used by the timer. 
      enum KindOfTime {
         SIMULATED_TIME = 0,
         OS_TIME
      };
Values:
  • SIMULATED_TIME : the timer will expired according to simulated time
  • OS_TIME : the timer will expired according to os time
typedef enum ScsTimer::ReSyncMode ScsTimer::ReSyncMode
Description:
enum corresponding to the computation type of the next wakeUp time. 
      enum ReSyncMode {
         NO_RESYNC = 0,
         RESYNC
      };
Values:
  • NO_RESYNC: the next wakeUp time will be current one increased by one period
  • RESYNC: the next wakeUp time will be current scadasoft time increased by one period.
typedef enum ScsTimer::WakeUpMode ScsTimer::WakeUpMode

         ONEWAY = 0,
         SYNC
      };
Values:

  • SYNC: the wakeUp calls will be twoway (synchonous).

)
Description:
constructor. It starts a timer with the specified characteristics. After timeout, the SCADAsoft library calls the expiration callback function with the arg argument. If period is not 0 (zero), the timer expires periodically with this value (specified in milliseconds). The first expiration time (tv) is specified as an absolute time, in seconds and microseconds.
Parameters:
tv[in]: timeval structure defining the first expiration time,
period[in]: period,
function[in]: user expiration callback function,
arg[in]: user optional argument passed as parameter of the callback when called.
reSync[in]: compute wakeuptime with the current Scadasoft Time (instead of previous wakeuptime).
wakeUpSync[in]: ask for oneway(0) or twoway(!=0) wakeUp (twoway is active only in redundant mode).
kindOfTime[in]: Scadasoft Time or OS Time (only in non redundant mode).
Return values:
none.
