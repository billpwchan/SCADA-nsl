Detailed Description

This class is one of the interface classes for the exported services of the RED SCADAsoft component (redundancy).

This class is used for the supervision of the processes of the current logical environment or of another logical environment. 

 #include "manager.h"

Description:
this function check periodically connection between the application and the name server running in the environment environment. If a change in this connection occurs, then callback is called with the status of the connection. " SCS_DOWN " means connection down whereas " SCS_UP " means connection up. ScsStatus AscManager::subscribeState ( const char *  logicalEnv,
const char *  processName,
char *&  state,
int &  subscriptionId,
MgrSupervisionCallback  notificationCallback,
ScsAny  arg = 0 
)
Description:
this function subscribe to the state of the process named processName running in the logicalEnv logical environment. The connection between the current application and the processName process is periodically checked. If a change in this connection occurs, then the notificationCallback callback function is called with the status of the connection: "SCS_DOWN" means the connection is down whereas "SCS_UP" means the connection is up and "SCS_INIT" means the connection is up, but no AscServer object was created. If the remote process has implemented an AscStateMgr object, the state "SCS_UP" will be replace by all state supported by the remote process. The current state of the process is also returned.
Parameters:
logicalEnv[in]: name of the logical environment in which the process is running,
processName[in]: process name,
state[out]: process current state,
subscriptionId[out]: subscription identifier,
notificationCallback[in]: user notification callback function,
arg[in]: user optional argument passed to the notification callback function when called.
Returns:
  • ScsValid : success.
  • ScsError : error.
  • AscORBError : error in orb communication.
