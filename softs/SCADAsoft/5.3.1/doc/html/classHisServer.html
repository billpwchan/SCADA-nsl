Detailed Description

This class allows the use of the management of archive services exported by the Historian Server.

This class is the interface class for the services exported by the HIS server. It is used for the management of archives. Archives are used to store the values of points as they change with time. One example of an application using archives is the graphical representation of variables in trends.

ScsStatus HisServer::createArchive ( const char *  name,

unsigned short  period = 0,
HisListType  listType = CIRCULAR,
unsigned short  maxFiles = 0,
unsigned short  capacity = 100,
HisArchiveType  type = RTAP_ARCHIVE 
)
Description:
this function creates an archive with the specified name. The created archive is composed of the attributes whose names are specified in the list attributeNameList. count is the number of attribute names in the list. The period parameter is set to 0 to create a non-periodical archive (default value) or to a non-null value to create a periodical archive (the units of the period are seconds). Non-periodical archives are updated each time the value of any of the attributes composing the archive changes, whilst periodical archives are updated at each expiration of the period.
Parameters:
name[in]: archive name,
size[in]: size of the data element list stored in the archive,
attributeList[in]: list of attribute names composing the archive,
count[in]: number of attributes in the archive,
period[in]: period of the archive (0 for a non-periodical archive, non-zero value for a periodical archive).
listType[in]: type of list used to store data for the archive, "LIMITED", "UNLIMITED" or "CIRCULAR".
maxFiles[in]: maximum number of disc files used to store data for the archive (0 for a non-collectable archive)
capacity[in]: number of entries per disc file.
type[in]: type of the archive, RTAP_ARCHIVE for an archive of values from the RTAP database, EXTERNAL_ARCHIVE for an archive updated by an external program.
Returns:
  • ScsValid : success.
  • HisArchiveCreationError : if an archive already exists with the same name,
  • HisRtapFunctionError : if an Rtap function has returned an error,
  • HisORBError : if an ORB exception was caught.
