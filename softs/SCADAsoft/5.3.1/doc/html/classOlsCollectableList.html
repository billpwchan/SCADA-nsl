Detailed Description

This class is one of the class of the OLS (Object List Server) SCADAsoft component.

This class is used to store data elements. An object (object of OlsData class) represents a set of basic data (object of OlsField class). An object list (object of OlsList class) groups a set of objects containing the same kind of information. The OlsCollectableList class behaves as OlsList except the number of OlsData within the list is limited and circular, and the list is periodically dumped into files on disk. The pathname parameter defines the directory on disk in which files are created. The maximum number of disk files for the list is defined by the maxFiles parameter and the maximum number of data in each file by the capacity parameter. When the number of elements inserted in the data list reaches capacity, these elements are dumped into a new file on disk. The list is circular: if the limit size is reached, new data overwrite the oldest data. The list should have a time field.

Description:
constructor. A list contains a set of data. Each data have the same fields called the fields of the list. The list manages subscriptions. The period is the period between two subscription refresh timeouts. The period is in milliseconds. The daily parameter specifies if all the data corresponding to the last day should be grouped into one global daily file. This concatenation happens at midnight and the daily files are located under a "Daily" subdirectory of the pathname directory. A non null value of daily validates this option and specifies the maximum of daily files on disk. When this maximum is reached and a new file is created, the oldest is removed.
