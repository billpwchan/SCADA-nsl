Detailed Description

Internal output (command) variable definition for the CTL component.

This class is used to describe a command to send to an equipment. A command corresponds to an internal output variable (IOV). 

 #include "ctlcmd.h"

constructor. This function creates a discrete internal output variable (DIOV) object.
Parameters:
name[in]: name of the output variable ("equipment_alias:iov_pointName"). The length of the name should not exceed CTL_K_MAXLEN_IOV_NAME characters.
value[in]: discrete value of the variable.
bypassInitCond[in]: defines the bypass mode for the init condition of the current commande. If the bypass mode is set (BYPASS value), the commande is sent even if the init condition is not correct. The default value is NO_BYPASS.
bypassRetCond[in]: defines the bypass mode for the return condition of the current commande. The default value is NO_BYPASS.
sendBehaviour[in]: defines the send behaviour when the return condition is already correct. The default value is to send the command.

Description:
constructor. This function creates a structured internal output variable (SIOV) object.
Parameters:
name[in]: name of the output variable ("equipment_alias:iov_pointName"). The length of the name should not exceed CTL_K_MAXLEN_IOV_NAME characters.
size[in]: size of the value. The size of the value should not exceed a maximum size (CTL_K_MAX_SIOV_VALUE_SIZE). If the value of the size parameter is bigger than this maximum size or if the buffer pointed by the value parameter contains more bytes than this maximum size, only a number of bytes corresponding to the maximum size are stored in the value.
value[in]: structured value of the variable. This parameter is a pointer to a buffer containing the structured value.
bypassInitCond[in]: defines the bypass mode for the init condition of the current commande. If the bypass mode is set (BYPASS value), the commande is sent even if the init condition is not correct. The default value is NO_BYPASS.
bypassRetCond[in]: defines the bypass mode for the return condition of the current commande. The default value is NO_BYPASS.
sendBehaviour[in]: defines the send behaviour when the return condition is already correct. The default value is to send the command.
