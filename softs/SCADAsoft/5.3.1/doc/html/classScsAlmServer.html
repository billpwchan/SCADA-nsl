Detailed Description

This class provides a remote access to the alarm server of SCADAsoft.

This class is used to remotely access the services of the alarm server of SCADAsoft. It allows to declare new alarms or events, and to acknowledge some alarms. 

 #include "scsalm.h"

Description:
this function declares a new (external) alarm or a new event.
Parameters:
extAlmId[out]: alarm identifier,
extAutoId[inout]: value automaton identifier (for entry mode, should absolutely be a negative value for an external alarm or a null value for an event). If the thresholdLow and/or thresholdLow parameter(s) is/are not null in order to define a range, the parameter is calculated by ScsAlmServer. In this case, it corresponds to the first free autoId found in the range.
state[in]: alarm state ("A" for alarm, "N" for normal or "T" for transitory),
valueAutomaton[in]: name of the value automaton,
ackAutomaton[in]: name of the ack automaton,
equipmentDate[in]:
acquisitionDate[in]:
severity[in]: alarm severity,
functionalCat[in]: functional category (applicative value), used for alarm synthesis,
geographicalCat[in]: geographical category (applicative value), used for alarm synthesis,
equipmentName[in]: equipment name,
message[in]: textual description of the alarm,
group1[in]:
group2[in]:
thresholdLow[in]: low threshold of the range, should be a negative value if used. Its default value is 0,
thresholdHigh[in]: high threshold of the range, should be a negative value if used. Its default value is 0.
Returns:
  • ScsValid : success.
  • ErrListAction : cannot update event list.
  • NullPointerAutomatonCreation : cannot create value automaton.
  • ErrNullPointerAckCreation : cannot create acknowlegement automaton.

  • ErrInternal : internal error.
  • ErrNullPointerAutomatonExt : cannot find value automaton.
