Detailed Description

External input variable (EIV) definition for the DAC component.

This class is used to describe a variable received from an equipment. The DacServer sends or receives external input variable lists. The variables can be of any type : int, float, bit fields. 

 #include "dac.h"

Member Enumeration Documentation

enum DacEIV::EivType
Description:
enum corresponding to the possible values for the type of an external input variable. 
      enum Status { 
         UNDEFINED_TYPE = 0,
         DEIV,
         AEIV,
         SEIV
      };
Values:
  • UNDEFINED_TYPE: unknown type, or not already set
  • DEIV: digital external input variable: integer.
  • AEIV: analogic external input variable: float.
  • SEIV: structured external input variable: bit field.
enum DacEIV::Status
Description:
enum corresponding to the possible values for the current status of an external input variable. 
      enum Status { 
         UNDEFINED = 0,
         VALID,
         INVALID_LINK,
         INVALID_INTERNAL,
         INVALID_RANGE,
         INVALID_GETTING,
         INVALID_CHATTERING,
         INVALID_INTERNAL_VALID,
         INVALID_INTERNAL_LINK,
         INVALID_INTERNAL_RANGE,
         INVALID_INTERNAL_GETTING,
         INVALID_INTERNAL_CHATTERING
      };
Values:
  • UNDEFINED: the status of the equipment associated with the variable is undefined.
  • VALID: the value of the variable is valid. The RTU associated to the variable is in the INS state and a value has been received from this RTU for the variable.
  • INVALID_LINK: data exchanges with the RTU associated to the variable are not possible. The value of the variable is the last received for the variable. Do not test directly this state, use isInvalidInternal() method.
  • INVALID_INTERNAL: the variable has been invalidated.
  • INVALID_RANGE: the value is wrong for the verification function.
  • INVALID_CHATTERING: the value is invalidated by the IO toggling detection algorithm.
  • INVALID_INTERNAL_VALID: variable invalidated with a previous state VALID.
  • INVALID_INTERNAL_LINK: variable invalidated with a previous state INVALID_LINK.
  • INVALID_INTERNAL_RANGE: variable invalidated with a previous state INVALID_RANGE.
  • INVALID_INTERNAL_GETTING: variable invalidated with a previous state INVALID_GETTING.
  • INVALID_INTERNAL_CHATTERING: variable invalidated with a previous state INVALID_CHATTERING.
