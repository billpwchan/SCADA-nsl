Detailed Description

This class is one of the interface classes for the exported services of the RED SCADAsoft component (redundancy).

This class is used for the supervision of the processes of the current physical environment. 

 #include "agent.h"

ScsStatus AscAgent::getProcessInfos ( char ***&  infos,
int &  nbprocess 
)
Description:
this function returns informations on n processes of ScsEnvTable.
Parameters:
infos[out]: char* matrix (12xn) with i varying from 0 to nbprocess-1 :
  • infos[0][i] : name of process
  • infos[1][i] : state of process (SCS_UP | SCS_INIT | SCS_DOWN...)
  • infos[2][i] : phase of process
  • infos[3][i] : termination of process (NONE | SHUTDOWN | RETRY)
  • infos[4][i] : notification of process (YES | NO)
  • infos[5][i] : shutdown order of process
  • infos[6][i] : supervision type of process (NONE | CORBA | UNIX | RTAP | PUSH)
  • infos[7][i] : priority of process (LOWEST | BELOW_NORMAL | NORMAL | ABOVE_NORMAL | HIGHEST)
  • infos[8][i] : run string of process
  • infos[9][i] : snapshot of process (NONE | BACKGROUND | FOREGROUND)
  • infos[10][i] : init timeout of process
  • infos[11][i] : pid of process (? | pid)
nbprocess[out]: number of process in ScsEnvTable
Returns:
  • ScsValid : success.
  • ScsError : error.
  • AscORBError : error in orb communication.

ScsStatus AscAgent::subscribeState ( const char *  processName,
int &  subscriptionId,
char *&  state,
AgtSupervisionCallback  notificationCallback,
ScsAny  arg = 0 
)
Description:
this function subscribe to the state of the process named processName running in the current physical environment. The connection between the current application and the processName process is periodically checked. If a change in this connection occurs, then the notificationCallback callback function is called with the status of the connection: "SCS_DOWN" means the connection is down whereas "SCS_UP" means the connection is up and "SCS_INIT" means the connection is up, but no AscServer object was created. If the remote process has implemented an AscStateMgr object, the state "SCS_UP" will be replace by all state supported by the remote process. The current state of the process is also returned.
Parameters:
processName[in]: process name,
subscriptionId[out]: subscription identifier,
state[out]: process current state,
notificationCallback[in]: user notification callback function,
arg[in]: user optional argument passed to the notification callback function when called.
Returns:
  • ScsValid : success.
  • ScsError : error.
  • AscORBError : error in orb communication.
