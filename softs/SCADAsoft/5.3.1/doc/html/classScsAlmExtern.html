Detailed Description

This class provides a remote access to the alarm server of SCADAsoft.

This class is used to remotely access the services of the alarm server of SCADAsoft. It allows to declare new external alarms or events. The characteristics of alarm classes are specified in a configuration file. 

 #include "scsalmext.h"

none.

ScsStatus ScsAlmExtern::notify ( long &  alarmId,
short  classId,
const char *  objectName,
short  objectId,
int  objectState,
va_list  args 
)
Description:
this function is an alternative method to declare a new (external) alarm or a new event. The number of parameters of this function is variable.
Parameters:
alarmId[out]: identification number (0-999) of the generated alarm,
classId[in]: identification number (0-999) of the alarm class (used to elaborate the unique external alarm identification number),
objectName[in]: object name (used as an equipment point alias) concerned by the alarm, if any,
objectId[in]: object identification number (should be specified only if no object name parameter was specified),
objectState[in]: object state (ALMEXT_OBJ_STATE_NORMAL or ALMEXT_OBJ_STATE_ERROR) determining which alarm attributes of the configuration file will be used,
args[in]: variables parameters used to construct the alarm message. Theses parameters correspond to the message format as defined in the alarm configuration file.
Returns:
  • ScsValid : success.
  • AlmBadParamError : the value of an input parameter is not correct.
  • AlmInternalError : no configuration data were found for specified alarm class.
  • AlmRtapFunctionError : error reading database using Rtap.
  • AlmScadasoftFunctionError : error reading database using DbmServer or calling notify() function of alarm server.
