1. Introduction

1.1 What is ANIMATOR

ANIMATORis an independent software module designed to create and animate interactive pictures for graphical man machine interfaces. It is made of an animation kernel able to load, save, display and animate large images of several thousands of objects.

Objects are high level graphical entities so that the user no longer deals with such elementary drawings such as lines, circles, ... but with meaningful and complex objects called symbols (equipment representations, symbolic information...). ANIMATOR features the AnStudio graphical editor that lets you easily design, save, and test high complex panels and symbol libraries.

ANIMATOR facilitates the development of man machine interface based on two-dimensional pictures. In particular, it reduces the development costs thanks to edition facilities and intuitive animation methods.

ANIMATOR Schematics is present in many application domains and is able to concentrate on specific features to comply with process supervision requirements.

1.2 ANIMATOR edition function

The ANIMATOR editors allow off-line creation and configuration of interactive two-dimensional pictures that will serve as support for the dialog with application software. These interactive images include elements of two different types:

         Interactive objects. They are animated either by animation variables that result from the application or by operator actions. A dynamic object can be set selectable, in which case a message will be sent to the application each time a selection event occurs on the object.

         Static objects. They together form the background of the image.

Dynamic objects provide generic animation capabilities such as moving, blinking, highlighting or changing visibility. Moreover, they are attached to graphical classes which define their graphical resources (such as colours or representations) and specific animation capabilities.

For example, an ANIMATOR class of the active symbol family can represent a circuit breaker in the electrical domain. It specifies the animation colours of the equipment and its different representations. Once completely defined, the class can be instantiated in images once or several times. By this way, it is possible to design interactive and meaningful symbol libraries, dedicated to specific domains, such as electricity distribution or rail transportation.

AnStudio is a graphical tool for creating and editing pictures. It allows you to design extensible graphical libraries for each application domain. Moreover, it enables to establish a link between the application variables and the image elements that are an animated representation of it.

1.3 Class edition

Several families of ANIMATOR classes are provided each of them having different specific animation capabilities. Every dynamic object contained into an ANIMATOR image will refer to a named class, which belongs to one of these predefined families. The object inherits from all the animation methods and graphical ressources of the class.

The different basic ANIMATOR class families are:

         active backdrop classes, for representing animated background objects,

         symbol classes, used as equipment representations,

         active symbols class, used to represent multi-representation objects,

         active number classes, used to display numerical values,

         active text classes, used to display textual information,

         filling symbol classes, used to display gauge and other fillable forms,

         mobile symbol classes,

To each class family, corresponds one specific graphical editor. These editors allow you to create, modify and save classes so as to instanciate them later through pictures.

 

1.4 Image edition

The AnStudio image editor is a graphical tool for creating and editing large interactive pictures. It first includes a typical edition function principally enabling:

         the creation of the background elements from basic graphic primitives such as lines or rectangles or by integration of digitised images,

         the creation of dynamic objects by instantiation of the ANIMATOR graphical classes,

         the manipulation of objects and graphic elements in the image (moving, copying, ...).

Moreover, the AnStudio editor includes a configuration function used to enter image and objects information so as to link them application variables. These information enable to automatically connect a picture and its components to the application and thus to animate them.

2. The AnStudio launcher

The AnStudio launcher is a general introduction panel to the ANIMATOR edition functions. It allows selecting the ANIMATOR workspace in which all the classes and images will be saved and launching the different class and image editors on the selected workspace.

2.1 Running AnStudio

The AnStudio editor is part of the ANIMATOR distribution media. To run AnStudio, do the following:

         Set the environment variable ANHOMEto the ANIMATOR installation directory.

         Set the environment variable ANPATHto the default workspace directory. This variable is not compulsory. Configuration files (animatot.ini) are stored into init, sub-directory of ANPATH(the launcher looks first the existance if this sub-directory; if init does not exists, the launcher looks at initialization files directly into ANPATH).

         Type the $ANHOME/anstudio/bin/<system>/anstudio.exe command where <system> stands for the system you are using, such as SunOS5.5 or HPUX.10.20.

Then the following panel is displayed at the topleft corner of the screen:

A popup-menu is available by clicking on the background of the launcher panel with the mouse right button. Using this menu, you can change of workspace, display the error list choose the current look and feel of the dialogs and set the current language (French or English).

2.2 Workspace manager

An ANIMATOR image workspaceis an independent database containing a graphical symbolic library dedicated to an application or a specific domain, and a set of images using this library. In addition to class and image description files, it also manages configuration filesthat specify all the graphical and animation resources used within the workspace (colours, fonts, interactors...). During edition, several different workspaces can be simultaneously opened.

To change the current workspace, click on the workspace selector icon from the AnStudio launcher (or choose Workspace from the popup-menu). The following panel then appears.

To set a workspace current, do the following.

         Select a workspace among the workspace lists or click on the Add button. A file selector box will appear and let you choose a new workspace directory.

         Click on the Broadcast button if you want to broadcastthe selected workspace to each open editor.

         Close the dialog box by clicking the Close button.

2.3 Error management

When an error or an invalid operation occurs while editing, a message is created and stored in an errorlist. This message is characterised by a severity and can be displayed in the error panel. To open this panel, select the Errors item from the launcher popup-menu.

All the error messages are displayed and can be sorted according to their severity. To remove all the messages from the list, select the Clean button.

2.4 Class editors

Every dynamic object inherits from an ANIMATOR class that defines generic graphical resources (such as animation colours or representations) and animation methods. These methods depend on the family to which the class belongs. For example, an active symbol class allows you displaying equipments with several representations whereas an active number class is rather used to display changing numerical values.

ANIMATOR provides a graphical editor per each class family . It allows creating and/or modifying classes and save them in an ANIMATOR workspace. To open a specific class editor, click on the corresponding icon from the AnStudio launcher.

2.5 Image editor

The ANIMATOR image editor allows creating and designing interactive pictures based on basic graphical elements and more complex and dynamic components. The main edition functions that it provides are listed below.

         definition of the image structure.

         creation and design of graphic primitives (labels, polylines, ellipses, ...),

         instantiation of ANIMATOR graphical classes within an image,

         creation of complex objects (groups and sub-images),

         connection of the dynamic image components to application data.

3. Symbol editor

A symbolis made of a group of graphical primitives (such as circles, arcs or texts) and enters in the composition of a more complex ANIMATOR class (such as mobile symbol) or of an image.

When used in a class, according to the family of the class, the symbol belongs to the dynamic or the static part of the class. For example, in the active symbol class family, symbols are used to represent the various shapes of the object during its animation (to each shape, corresponds a symbol). When used in an image, the symbol is part of the background.

The purpose of this chapter is to present the way to create, modify and save a symbol class.

3.1 Launching the symbol editor

To launch the symbol editor, just click on the symbol icon from the AnStudio launcher:

The symbol editor contains the following elements:

         a general menu bar,

         a work area, used edit the symbol representation,

         an horizontal tool bar, underneath the menu bar, providing shortcuts to the most useful commands,

         an inspector area used to edit the properties of graphic objects,

         a vertical tool bar located on the left of the main window, providing a set of drawing tools.

3.2 Creating/opening a symbol

To create a new symbol, select New from the File menu or click on the New icon from the main tool bar. The symbol editor is re-initialised so as to let you define a new class.

To open an existing symbol, select Open from the File menu or click on the Open icon in the main tool bar. A file selector box appears and lets you choose a symbol file in the symbol directory of the current ANIMATOR workspace.

To preview the selected symbol, turn on the Show toggle. Filters are also available on file and directory names so as to facilitate the search of a particular class. The selected symbol is loaded in the symbol editor and its graphical representation is displayed in the work area.

3.3 Editing a symbol

Editing a symbol principally consists in drawing its representation from basic graphical primitives (such as arcs, circles or texts). The symbol editor proposes a set of drawing tools contributing to facilitate the drawings of these primitives. Moreover, it provides commands allowing modifying easily and accurately symbol properties: geometry (size, position, alignments...), resources (colours, character fonts, line width...) and class information.

3.3.1 Drawing tools

The vertical tool bar located to the left of the editor window provides a set of editing modes. These modes let you select an object in the editor workspace and create basic graphical primitives such as lines, circles, or labels.

Selection

The selectionmode is used to select, drag and resize a graphic object in the work area. Selection is realised by clicking on the object or by dragging a rectangle on an area. Use Shift-Click to do multi-selection. If two objects are superposed, the first click selects the forward object; the second click the second object and so on. This mode is automatically activated when the symbol editor is launched.

Polyselection

Use this mode to move or rotate the different points of a polyline, polygon, spline, filled spline or closed spline. To complete the operation, double click in the work area or select another mode. Adding a new point is also possible by clicking on a point handle with the Control + middle mouse button. To delete a point, select its handle and do Control + right mouse button click.

Lines

Use this mode to draw a simple line. Press the left mouse button to indicate the first point, drag the cursor to the second point and finally release the mouse button to complete the operation.

Polylines

Use this mode to draw polyline objects. Click in the work area to indicate the points in the order you want them to be inserted in the polyline, and double-click to complete the operation.

Arcs

Use this mode to draw an arcobject. First drag a rectangle to indicate the bounding box of the arc. Then drag the resize point and/or the rotate point, respectively to indicate the angle range and/or the start angle. Finally, double-click to complete the operation.

Filled arcs

Used this mode to draw a filled arc as the same way as described above for the arcs object.

Polygons

Use this mode to draw polygons. Click in the work area to indicate the points in the order you want them to be in the polygonand double click to complete the operation.

Splines

This mode lets you draw curved lines in form of Bzier splines. Click in the workspace to indicate the points in the order you want them to be inserted in the spline, and double-click to complete the operation.

Filled splines

Use this mode to draw a filled spline in the same way as described above for the splines objects.

Closed splines

Use this mode to draw a closed spline in the same way as described above for the splines objects.

Rectangles

This mode lets you create simple rectangles. Press the left mouse button in the work area to indicate the first corner of the rectangle, drag the cursor to define its bounding box and finally release the mouse button to complete the operation.

Filled rectangles

Use this mode to draw filled rectangles in the same way as described above for rectangles objects.

Ellipses

This mode lets you create ellipses. Press the left mouse button in the work area to indicate the position of the ellipseand drag the cursor up to define its bounding box. Finally release the mouse button to complete the operation.

Filled ellipses

Use this mode to create filled ellipses in the same way as described above for ellipses objects.

Circles

Use this mode to draw a circlein the same way as described above for ellipses objects.

Labels

This mode lets you create single lines of texts. Click in the workspace to indicate the labelposition, type a string, and then press Return to complete the operation. To edit an existing label, double-click on it. This action opens a dialog that lets you change the label contents and the character font.

Origin marker

Use this mode to draw an origin markerin the same way as described above for rectangle objects. The marker (in a form of a red cross object) indicates the origin of the symbol. Therefore, inserting a symbol instance at the position (x, y) within an image makes the symbol origin match to the point (x, y).

 

An object creator mode (such as Labels or Lines) is automatically de-activated after creating the object. When a mode is terminated, AnStudio automatically returns to the default mode. For example, open the symbol editor and do the following sequence:

         select the label mode,

         click anywhere in the work area to display the label box,

         type a string in the label box,

         press Return.

The Labels mode is de-activated and the Selection mode is automatically selected. If, however, you want to set a particular mode as default, you can do so by clicking its icon twice. This is handy if you want for example to create several labels, since if you clicked on the Label icon twice, the Labels mode remains selected. This means that when youve typed the string in the label box and pressed Return, you can create another label by clicking again in the drawing workspace.

3.3.2 Graphical resources

To modify the graphical resourcesof an object, select it in the drawing workspace and choose the entry Resources from the Tools menu (or click on the Resources icon in the main tool bar). It opens the Resources panel.

Below is the description of the various elements that make up this panel.

Foreground

The current foreground colouris displayed in the rectangle to the right of the Foreground label. Clicking on this rectangle opens a colour selector that lets you choose a new colour. All the colours defined in the current ANIMATOR workspace are listed in this dialog box. When selecting a colour, its application name is displayed at the bottom of the panel.

Background

The current background colouris displayed in the rectangle to the right of the Background label. Clicking on this rectangle opens a colour selector that lets you choose a new colour.

Font

The name of the current character fontis displayed in the combo box to the right of the Font label. This combo box lets you choose another font, from those defined for the current ANIMATOR workspace.

Fill rule

Fill rule defines which points are to be considered as inside a filled polygon, depending on the count of crossing segments that defines the shape of the area to be filled. The combo box to the right of the Fill Rule label displays the name of the current fill ruleand lets you modify it.

Arc mode

Arc mode defines the way in which filled arcs are to be drawn. The current arc modeis displayed in the combo box to the right of the Arc mode label. This combo box lets you choose another arc mode from those available.

Pattern

The current patternis shown in the rectangle to the right of the Pattern label. The combo box to the right of this rectangle displays its name and lets you choose another pattern.

Line style

The current line styleis shown by the horizontal line to the right of the Line style label. The combo box to the right of this line displays its name and lets you choose another style.

Line width

The current line widthis shown by the horizontal line to the right of the Line width label. The combo box to the right of this line displays its value and lets you choose or enter another line width.

Fill

The current filling stateis displayed in the combo box to the right of the Fill label. This combo box lets you change the filling state of an object

 

When an object is selected, the Resources panel, if opened, is automatically updated with the current resources of the selected object. To modify the properties, change one or several resources and Apply. The selected object is then refreshed in the work area.

If several objects are selected, the Resources panel displays only the common resources of the selected objects (the other fields are empty). To apply any change to all the objects, turn on the toggles located to the right of the resources that have been changed, and then Apply.

3.3.3 Alignments

To align objects in the symbol editor work area, select them all and choose Alignments from the Tools menu. This command opens and raises the Alignmentpanel.

This panel contains the following elements.

Left alignment

Moves the selected objects horizontally so that their left borders are vertically aligned.

Right alignment

Moves the selected objects horizontally so that their right borders are vertically aligned.

Vertically centred

Moves the selected objects horizontally so that their centres are vertically aligned.

Same width

Resizes the objects so that they have the same width.

Top alignment

Moves the selected objects vertically so that their top borders are vertically aligned.

Bottom alignment

Moves the selected objects vertically so that their bottom borders are vertically aligned.

Horizontally centred

Moves the selected objects vertically so that their centres are horizontally aligned.

Same height

Resizes the objects so that they have the same height.

Vertical spaces

Creates an equal amount of vertical spaces between the selected objects.

Horizontal spaces

Creates an equal amount of horizontal spaces between the selected objects.

3.3.4 Geometric transformations

To change the geometryof an object (i.e. its size, its position and its orientation), use the select mode. You can by this drag the object or resize it by clicking on one of its selection handles. More accurate geometric transformations can be carried out by using the Move/Rotate/Resize panel (choose Move/Rotate/Resize from the Tools menu).

Below is the description of the elements that make up this panel.

Move

The Move number field lets you specify the translation distance (in number of pixels). To apply the translation on an object, select it in the symbol work area and click on one of the four arrow buttons.

Rotate

The Rotate number field lets you specify the rotation angle. To apply the corresponding rotation on an object, select it in the symbol work area and click on a rotate arrow button.

Scale

The Scale number field lets you specify the homothetic transformation factor. To apply the corresponding transformation on an objet, select it in the symbol work area and click on a scale arrow button. Turn on the horizontal and/or vertical toggles to specify whether the transformation is applied on the x and/or y coordinates.

3.3.5 Bitmaps

The AnStudio symbol editor mainly handles vectorial entities, that is, shapes made of lines and curves that can easily be manipulated to change their visible aspect in terms of geometric characteristics. However, you can also import image bitmapsin the symbol work area. The supported formats of bitmaps are BMP, PNG, XPM, PBM, PGM and PPM.

To import a bitmap, choose Import bitmap from the Tools menu. A file selector box appears and proposes you all the bitmap files that are present in the icon directory of the current ANIMATOR workspace. To insert a bitmap in the symbol work area, select its description file and then apply.

3.3.6 Grouping and ungrouping

The symbol editor provides a grouping function that enables to create composite objects. To create a group, select all the objects you want to insert in the group and choose Group from the Processing menu. A single composite object is then created, containing all the previous selected objects, and becomes selected. It forms an atomic element so that it can be handled like every other basic primitive (in moving, resizing, changing graphical resources...).

Groups are recursive, i.e. groups can contains other groups.

To ungroup a composite element, select it and choose Ungroup from the Processing menu.

Note that the symbol editor automatically groups all the graphical objects that define the symbol representation when saving it. Therefore, just after opening a symbol class, you have to ungroup the representation to handle the different components individually.

3.3.7 Miscellaneous

Some others utilities are available to facilitate the design of symbol representations, such as Copy/Pasteor Undooperations. They are linked in general to keyboard accelerators.

Undo

To cancel the last operation performed on a primitive or a set of primitive, choose Undo from the Edit menu. This action does not allow cancelling the choice of options in a dialog box, but only primitive handling, including deletions, cut/paste actions, moving and resizing.

Copy/Paste

To perform Copy/Paste operations on graphic objects, select them in the symbol workspace and choose Copy from the Edit menu. The selected primitives are stored in a press for later pasting. Now choose Paste from the Edit menu to display the primitives previously copied. Without being cleared, the content of the press is copied in the symbol work area.

The Cut action has the same effect than the Copy one, except that the selected primitives are removed from the symbol work area.

Superposition

To raise or to lower a graphic object, select it in the symbol work area and choose Put to front or Put to back from the Edit menu. The selected object is put respectively forwards or backwards the other elements. To select a backward element, click several times on its bounding box until the selection handles appears around it.

Gauge

You can transform the symbol being edited into a gaugeor a filling symbol (used to design a filling symbol class). This operation can only be performed with a polypoint object: a polyline, a polygon, a rectangle, an ellipse or a spline. To do so, select the object and choose Transform to a filling symbol from the Processing menu.

3.4 Managing work area

The symbol work area characteristics can be customised so as to help you to create and edit symbol representations.

The View/Grid panel allows you to specify its main characteristics such as size, grid, and background. To open it, choose the View/Grid entry from the Tools menu.

3.4.1 Grid

The Gridsection lets you define a point grid to be displayed in the symbol work area so as to facilitate object alignments. It contains the following elements.

Origin

Use the x and y fields to change the grid origin.

Spacing

Use the x and y fields to set the spacing between the grid points.

Show

Use the x and y fields to specify the number of points drawn per grid step.

Active

Turn on this toggle to activate the grid step.

Visible

Turn on this toggle to make the grid visible.

3.4.2 Window characteristics

Using the View/Grid panel, you can also change the size and the background of the symbol work area. The elements contained in the Window section of the panel are listed below.

Width

Use this field to specify the width of the current work area.

Height

Use this field to specify the height of the current work area.

Background

The box to the right of the Background label displays the current background colour of the work area. To select another colour, click on the Select button.

 

Another way to change the size of the work area is to choose the entry Fit to contents from the Edit menu. This action opens a dialog box that lets you set a margin to resize the workspace so that the right most and the bottom most objects are completely visible along with the margin you set in the dialog box.

3.4.3 Panning and zooming

The Zoom in and Zoom out icons located in the main menu bar of the symbol editor lets you perform zoomoperations in the symbol work area. Keyboard shortcuts are also available: the PageUp and PageDown keys allows you respectively to zoom in and zoom out and the arrow keys let you translate the work space view.

Pressing the End key applies a view transformation so that all the objects are visible in the work area.

3.5 Saving a symbol

To save the symbol being edited in the workspace, choose Save from the File menu or click on the Save icon from the main tool bar of the symbol editor. To create a new ANIMATOR image, select New from the File menu or click on the New icon located in the general tool bar. This action opens a new window that stands for the image frame. In this frame, you will process drawing operations, and import class instances.

To open an existing image, select Open from the File menu or just click on the Open icon in the main tool bar.

11.3 Layers

ANIMATOR images are organised into layers. A layercorresponds to a storage place, which contains several graphic objects according to specific application criteria. For example, an escalator image may contain a mechanical layer, where will be inserted all the mechanical equipment and an electrical layer, which will represent the electrical part of the escalator.

All the layers are named and are defined in the workspace configuration files. They are ordered (the mechanical layer is always drawn in front of the electrical layer for example), and all the objects located within the same layers are ordered too.

11.3.1 Adding/removing a layer

To add a new layer into an image, select Layers/Levels from the Tools menu or click on the Layers/Levels icon located in the general tool bar of the image editor. Click on the Layers tab to display the following page.

This page displays the list of the layers currently present within the image. The first column shows the layer names, the second their states (V=visible / NV = Non Visible and A = active / NA = Non Active) and the third the layer comments.

To add a new layer, click on the Add button. A layer selector appears and lets you choose a layer from those defined in the current ANIMATOR workspace. To remove a layer, select it in the layer list and click on the Remove button.

11.3.2 Layer states

Image layers are characterised by visibility and sensitivity states. To set a layer visible or not visible (that make all the objects contained in this layer respectively visible or not visible), select it and click on the Visible or Invisible button.

To set a layer sensible or not sensible (that makes all the objects contained into the layer respectively to react or not react to designation events), select it and click on the Active or Inactive button. Note that the state of the layer is automatically updated in the second column of the layer list

Since a simple object cannot be placed in two layers simultaneously, the image editor manages a current layer, where all the new created objects are inserted during edition. This layer can not be declared invisible or in- sensitive. To set a layer current, select it in the layer list and click on the Current button.

11.4 Decluttering levels

Decluttering levels define visibility scopes through zoom factors. An object associated to a decluttering levelwill only be visible in the range of a zooming factor specified in the decluttering level.

For example, lets consider a large image representing a complex electrical network. When the entire network is visualised (i.e. at a low zooming factor), only schematic representations of the equipment or stations are displayed. When zooming into the image, these simplified components are gradually replaced with more complex and detailed objects. Equipment labels or low voltage electric lines appears. To build this gradued mechanism, several decluttering levels are necessary within the image.

In conclusion, decluttering levels allows managing the visibility of the image components according to the zooming factor. They are named and defined in the workspace configuration files.

11.4.1 Adding/removing a decluttering level

To supply the current image with a new decluttering level, select Layers/Levels from the Tools menu or click on the Layers/Levels icon located in the general tool bar of the image editor. Click on the Levels tab to display the decluttering levels page.

This page displays the list of all the decluttering levels currently present within the image. The first column shows the level names, the second their states (V=visible / NV = Non Visible / Z = Zoom Sensible) and the third the decluttering level comments.

To add a new decluttering level, click on the Add button. A level selector appears and proposes you all the levels that are defined within the current ANIMATOR workspace. Select one level and then apply. The item selected appears in the decluttering level list.

11.4.2 Visibility states

Specific states can be associated to a decluttering level to provide edition facilities:

Visible

All the objects associated to the level are visible, whatever the current zooming factor.

Invisible

All the objects associated with the level are invisible, whatever the current zooming factor.

Zoom sensible

It corresponds to the normal state of decluttering.

 

To change the visibility state of a decluttering level, select it in the level list and turn one of the state toggles to the right of the level page. This action automatically updates the level state in the list.

Since only one decluttering level can be affected to a graphic object, the image editor manages a current level that is set by default to all the new objects inserted in the image. This level can not be declared invisible. To set a decluttering level current, select it in the level list and click on the Current button.

11.6 Class processing

The previous chapter explained you how to insert static graphic primitives within an image. This one describes how to import ANIMATOR class instancesin the image so as to compose its dynamic part.

To create a dynamic object, select Import/Transform from the Processing menu or click on the corresponding icon from the general tool bar of the image editor. This action opens the following panel.

This panel contains:

         a class selector proposing all the classes contained in the current ANIMATOR workspace,

         a class previewer used to give a simple graphic representation of the selected class,

         a transformation tool used to create the specific active backdrop objects.

11.6.1 Importing an instance

To import a graphical class instance into an image (except for active backdrop classes), do the following.

         Select a class using the class selector. To do this, choose first the class family (except Active Backdrop) using the combo box located at the top of the panel. Turn on the Show toggle to preview the selected class. Filters are available on file and directory names so as to facilitate the search of a particular item.

         Drag the class representation from the class previewer and drop it into the image buffer. A new instance of the selected graphical class is created and inserted in the image.

11.6.2 Transforming a graphic primitive

The creation of an Active Backdrop instance consists in affecting an active backdrop class to a graphic primitive contained into an image. This primitive is transformed into a dynamic object (and especially inherits from the animation colours of the active backdrop class).

To create an active backdrop object, do the following:

         Create a graphic primitive (or a primitive set) in the image buffer and select it.

         Select an active backdrop class with the Import/Transform panel. Click on the Uniform tab in the transformation notebook. The graphic primitive is displayed with red colour in the notebook page.

         Click on the Transform button. The graphic primitive becomes an active backdrop object.

To undo the transformation from a graphic primitive to an active backdrop object, select the active backdrop and choose Undo SAOR from the Processing menu.

11.6.3 Transforming a graphic object

The transformation tool allows you to transform a graphic primitive into any ANIMATOR class instance. Only for active backdrop class, the representation of the original primitive is preserved. To do such a transformation, perform the same sequence as for active backdrop.

For example, to transform a mobile symbol object into a active symbol object:

         select the mobile symbol from the image buffer,

         select the active symbol class using the class selector from the Import/Transform panel,

         click on the Uniform tab and select the Transform button. The mobile symbol is then changed into an active symbol object.

The Apply all toggle allows to apply transform operation on all objects owning the same class name. The Keep proportions toggle allows to preserve the position and size of the object to be transformed for the replaced one.

The transform operations can also be performed on a selection of several objects, using the Composite page of the transformation tool.

11.6.4 Sub-images

Once an image has been designed with the image editor, it can be itself instantiated within another image, like a standard class. This mechanism allows you to define once for all complex dynamic objects and instantiate them in other images. The content of the sub-imageis not duplicated, so that any modification on a sub-image will take effect on each instance of it.

Note that the layer structure of images is always preserved. Thus, when inserting a sub-image in another image, all the objects contained in a given layer of the sub-image will be automatically drawn in the same layer of the parent image. Its the reason why the instantiation of a sub-image may sometimes insert new layers in the target image.

To import a sub-image in an image, juts proceed as described for basic classes. Choose Import/Transform from the Processing menu of the image editor. Select a sub-image with the class selector of the Import/Transform panel and finally drop it in the target image. The new instance can be now handled like any other graphic objects (in terms of resizing, moving...).

11.6.5 Designation

The Designate page (in the main image editor notebook) displays the semantic information attached to an image object. These properties are used to identify the selected object and eventually to connect it to the application (real-time database for example).

The designationpage contains the following elements.

         a class section identifying the graphic class from which inherits the selected object. The Type filed displays the class family (such as Active Text, Active Symbol or Bargraph) whereas the Name field presents the name of the class,

         an object section that allows you to define a name and a comment for the selected object. The Name and Comment toggles allow you to particularise in the image buffer all the objects that have respectively a name or a comment (by using the selection colour),

         a designation area that allows you to set the selected object sensitive or not. A non-sensitive object never receives designation events.

11.7 Selections

Many commands are available in the image editor so as to manipulate the static or dynamic objects that make up the image (moving/resizing, grouping, alignment and so on). In this context, selectionoperations are fundamental to carry out these operations. Several selection modes are available and are described in this chapter.

11.7.1 Simple selection

To select a graphical object, just click on the image buffer in the bounding box of the object. Handles squares then appear around this object, allowing you to move and resize it. The properties of the selected objet are also displayed in the message area.

If some objects are superposed, a first click selects the forward object; a second click selects the backward object and so on. Note that you can deactivate selection in a particular layer (using the Layer panel) so as to avoid selecting some of the image objects.

11.7.2 Multiple selection

To select sveral objects, use Shift + click. To select all the objects contained in a given region of the image, click on the image buffer, drag the mouse pointer and release the selection button. All the objects contained into the dragged rectangle will be selected.

An other way to select multiple objects is to use the selection tool. This tool allows applying some criteria on the object selection (such as the class type, the class name or the layer location). To open it, select Add to selection from the Edit menu. The selection panel contains a notebook with three pages, each of them proposing selection criteria.

The options in the Class page let you enter object type criteria. For example, enter a class name in the Class name field and then click on the Apply button. All the objects that inherit from the class youve entered the name, become selected in the image buffer. The Class type combo box lets you choose a class family (such as active text or symbol). The Element type combo box is dedicated to static background objects. It allows adding a selection criterion on graphic primitive types (for example to select all the lines or all the labels contained in the image).

The options in the Object page let you specify object criteria. For example, enter an object name in the Object name field and press the Apply button. All the corresponding objects become selected in the image buffer. The Object comment field lets you identically select objects according to their comments.

The options in the Layout page lets you define layer and level criteria. Choose a layer and/or a decluttering level using respectively the Layer and Level combo box and press the Apply button. All the objects contained in the mentioned layer and/or attached to the decluttering level become selected in the image buffer.

11.8.3 Superposition

The objects contained in the same image layer are ordered. When inserting a new graphic object in a layer, it is appended at the end of the drawing list, i.e. becomes the last object to be drawn. To change the drawing order of an object, select it in the image buffer and choose Put to front, Put to back, Forward or Backward from the Edit menu of the image editor.

The Put to front command increments the index of the object in the drawing order of the layer, whereas the Forward action put the object front to all others object located in the same layer.

Note that the layers contained into an image are themselves ordered. Thus if a layer A is front of a layer B, all the objects contained in the layer A are drawn in front of the layer B. This layer drawing order is defined in the ANIMATOR workspace configuration files.

11.9 Grouping and ungrouping

Two kinds of groups exist in ANIMATOR image: the primitive sets and the dynamic object set. A primitive set only contains graphic primitive and is quite identical to a simple primitive. It can be placed in any layer and decluttering level, and changing its resources (such as colour for example) affects all its components.

A dynamic object set contains at least one dynamic object. It is always attached to the AnAlwaysVisibleLevel predefined decluttering level. However, its components can be individually associated to a specific level.

To groupseveral objects together, select them and choose Group from the Processing menu. A single composite object is then created, containing all the previous selected objects. This new object can be now handled like every other simple primitive (in terms of resources or geometry...). Groups are recursive, i.e. groups can contains other groups.

To ungroup a composite element, select it and choose Ungroup from the Processing menu.

11.10 Aggregates creation

During an image design, the end-user can select some objets, class instances or sub-images instances and decide to store them into a file called aggregate.

So, the option Create aggregate, under the main menu File, allows to define the name and a comment for a new aggregate:

The aggregate files are stored into a specific directory called agregat under the image directory.

11.14 Import DXF

The AnStudio editor is able to import DXFschematics and to transform it into ANIMATOR images. The following rules are applied:

         All DXF elements are transformed into ANIMATOR objects and inserted in the same layer.

         DXF elements are converted into static background primitives, except named blocks that are transformed into symbol or active symbol objects. For each block differently named, an ANIMATOR class is created and saved. Each DXF block is then transformed to an instance of the ANIMATOR class which name corresponds to the block name.

         All labels are converted using the same character font.

Colours are not all conserved since ANIMATOR colours defined into the current workspace does not necessarily correspond to the DXF ones.

         An homothetic transformation can be applied during the import operation to all the DXF objects (if their co-ordinates are too large).

To import a DXF diagram, choose Import DXF from the general File menu. This action opens a file selector that lets you choose a DXF file. Select a DXF file and apply. A new image is then created and is completed with the transformed DXF elements. All the classes that are created by the import operation are saved in the DXF/<name> directory, where <name> is the DXF file name.

Use the DXF page of the Option panel to set the import DXF parameters (choose Options from the Tools menu to open the panel). This page contains the following elements.

Origin

Use the x and y fields to specify the origin of the new image.

Font/Scale

The combo box under the Font/Scale label displays the name of the character fontused to import DXF labels. The number field to the right of this combo box displays the value of the scale that will be applied to DXF labels.

The main panel is organized into four areas, specific to the animable objects classes:

         The General properties area deals with any type of animable object. The check boxes indicate whether the animations have to be applied to the current selection or not:

         Visibility: Toggles the objects visibility state.

         Alarm: Toggles the objects alarm state.

         Highlight: Toggles the objects highlight state.

         Blinking: Toggles the objects blink state. A combo box allows you to select the blinking period.

         Scale: Applies a local zoom factor to the selected objects. A combo box allows you to select the zoom factor to be applied. The Default choice restores the objects original dimensions.

         Background color: applies a background color among those defined for each of the selected objects.

         Foreground color: applies a foreground color among those defined for each of the selected objects.

         Force color: forces the foreground color of all the selected objects. The associated button leads to a color selection box (see 3.3.2 Graphical resources). This option has priority over the foreground color number. Thus, the Foreground color check box is disabled when the Force color one is checked.

         The Active symbol objects properties area refers to the ANIMATOR active symbol objects. It contains a check box that controls whether a symbol number has to be applied to the selected active symbol objects. If checked, a combo box lets you select the symbol number you want to be applied to the selection. This combo box is editable, so that you can type a symbol number whith the keyboard.

         The Active text objects properties area refers to the ANIMATOR active text objects. It contains a check box that controls whether a label has to be applied to the selected active text objects. If checked, the objects default text value will be applied to the selection and the Text field is disabled . If not, the Text field is editable and you can type the label you want to be applied to the selected active text objects.

         The Active number, mobile symbol, filling symbol & bargraph objects properties area refers to objects that have numerical attributes, such as active numbers, mobile symbols, filling symbols and bargraphs.

         If the Force default value box associated to the Value number field is checked, this field is disabled, and the default value for V1 (hight value) is applied to the selected objects. If not, the Value field is enabled and you can type the value you want to be applied.

         If the Force default value box associated to the Low Value number field is checked, this field is disabled, and the default value for V2 (low value) is applied to the selected objects. If not, the Low Value field is enabled and you can type the value you want to be applied. This property only deals with filling symbols and bargraphs.

11.17 Miscellaneous

11.17.1 Compilation

ANIMATOR images can be compiled so as to create a single binary file that contains of course the image description but also the description of all the classes used in the image. This compilationallows moreover improving performances when reading and loading images from the ANIMATOR workspace.

To compile the image being edited, choose Compile from the File general menu. The binary file is then saved in the binary directory of the ANIMATOR workspace. To compile all the images contained into the current ANIMATOR workspace, select Compile all from the File menu.

11.17.2 Postscript

A dump function is available so as to print the contents of an image into a text file that can be immediately printed on a PostScriptprinter. To generate this file, choose Export/PostScript from the general File menu.
